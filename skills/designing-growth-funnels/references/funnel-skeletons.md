# Funnel Skeletons

Use this reference when choosing the primary and fallback funnel path.

| Skeleton ID | Use when | Primary KPI | Core events |
| --- | --- | --- | --- |
| `problem_aware` | Cold traffic with low awareness | `Brief Started / Landing Viewed` | `Landing Viewed`, `Problem CTA Clicked`, `Brief Started`, `Brief Completed`, `Diagnosis Lite Viewed` |
| `diagnostic` | User knows the pain but not the fix | `Roadmap Viewed / Diagnosis Generated` | `Brief Started`, `Brief Completed`, `Diagnosis Generated`, `Roadmap Viewed`, `Trial Started`, `Demo Booked` |
| `trial_to_value` | Self-serve first value can happen quickly | `First Value Reached / Trial Started` | `Trial Started`, `Onboarding Started`, `Source Connected`, `First Value Reached`, `Paywall Viewed` |
| `direct_offer` | High-intent product-aware user | `Trial Started / Landing Viewed` or `Payment Completed / Pricing Viewed` | `Landing Viewed`, `Pricing Viewed`, `Trial Started`, `Checkout Started`, `Payment Completed` |
| `demo_led` | High ACV, multiple stakeholders, or slow setup | `Demo Booked / Qualified Visit` | `ROI Calculator Used`, `Qualification Completed`, `Demo Booked`, `Pilot Started`, `Deal Won` |
| `creator_subscription` | Recurring content, community, or access value | `Subscription Started / Personalized Offer Viewed` | `Fit Quiz Completed`, `Path Generated`, `Free Unlock Viewed`, `Subscription Started`, `Week 1 Active` |
| `expansion_rescue` | Existing customer expansion, rescue, or reactivation | `Upgrade Started / Health Check Viewed` or `Reactivation / At-Risk` | `Health Check Viewed`, `Gap Diagnosis Generated`, `Action Pack Started`, `Upgrade Completed`, `Week Return` |

## Skeleton standard

Every skeleton output must include:

- primary path;
- fallback path;
- one primary KPI;
- one guardrail;
- key friction risk;
- first value target;
- handoff rule for assisted paths.

# Screen Specs

Use this reference when producing screen-level specs or microcopy.

## Belief-shift rule

Each screen should change exactly one target belief. Avoid screens that mix education, proof, setup, pricing, and support without a single conversion job.

## Screen matrix

| Stage | Target belief | Show | CTA | Metric |
| --- | --- | --- | --- | --- |
| Landing | This is about my situation and worth continuing | pain/outcome, proof, channel match, result preview | Check my path / Start diagnosis | `Brief Started / Landing Viewed` |
| Brief | The system understands my context | 3-7 adaptive questions, progress, why-this-asks | Continue diagnosis | `Brief Completed / Brief Started` |
| Diagnosis | The problem is specific and fixable | ranked gaps, severity, impact, confidence | Build my plan | `Roadmap Viewed / Diagnosis Generated` |
| Roadmap | I have a credible route to first value | quick win, 7/30/90 path, effort/impact | Start / Book review | `Onboarding Started / Roadmap Viewed` |
| Onboarding | I can get value without a tutorial | one task at a time, prefill, contextual help | Create first result | `First Value Reached / Onboarding Started` |
| Trial | I already felt value and paid mode extends it | live output, usage milestones, next action | Unlock full mode | `Active Trial Users / Trial Started` |
| Paywall | Payment preserves and extends progress | outcome plans, proof, FAQ, preserved state | Choose plan | `Payment Completed / Checkout Started` |
| Retention | Returning produces new value | digest, benchmark, one action, progress | Run this week's improvement | D7/D30 retention |

## Output format

For each screen, include:

- stage;
- target belief;
- user question;
- content modules;
- primary CTA;
- secondary CTA if justified;
- microcopy;
- event fired;
- success metric;
- guardrail;
- blocked assumptions.
